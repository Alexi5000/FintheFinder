# Skills And Tools

Fin separates research judgment from executable capability.

| Kind | Purpose | Example |
| --- | --- | --- |
| Skill | Procedural playbook, review standard, or decision policy | Source-quality policy, citation audit rubric, HITL approval checklist |
| Tool | Typed executable function with bounded side effects | Exa web search, result evaluation, learning extraction, contract sync |
| Agent | Role-specific model behavior that uses skills and tools | Planner, source evaluator, contradiction checker, final reviewer |

Runtime work should keep tools small, typed, and observable. Skills should be versioned prose that an agent can cite in run events and eval traces.
